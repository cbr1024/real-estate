## 1. 개발 환경 세팅

### 1.1 사전 준비

| 소프트웨어 | 버전 | 용도 |
|-----------|------|------|
| Docker Desktop | 최신 | 전체 서비스 컨테이너 관리 |
| Git | 최신 | 소스코드 버전 관리 |
| VS Code (권장) | 최신 | 코드 편집기 |

> Docker Desktop만 설치하면 Node.js, Python, PostgreSQL 등을 별도 설치할 필요 없음

### 1.2 프로젝트 클론 및 환경변수 설정

```bash
# 1. 프로젝트 클론
git clone https://github.com/<your-repo>/real-estate.git
cd real-estate

# 2. 환경변수 파일 생성
cp .env.example .env
```

`.env` 파일을 열어 아래 항목을 실제 값으로 변경:

```env
# PostgreSQL
POSTGRES_USER=apartment_admin
POSTGRES_PASSWORD=<강한 비밀번호>
POSTGRES_DB=apartment_db
POSTGRES_APP_PASSWORD=<앱용 비밀번호>

# Redis
REDIS_PASSWORD=<Redis 비밀번호>

# 네이버 지도 API (필수)
NAVER_MAP_CLIENT_ID=<네이버 클라우드 플랫폼에서 발급>
NAVER_MAP_CLIENT_SECRET=<같이 발급되는 시크릿>

# 네이버 검색 API (학군 정보용)
NAVER_SEARCH_CLIENT_ID=<네이버 개발자센터에서 발급>
NAVER_SEARCH_CLIENT_SECRET=<같이 발급되는 시크릿>

# 공공데이터포털 API (실거래가 수집)
DATA_GO_KR_API_KEY=<data.go.kr에서 활용신청 후 발급>

# JWT 인증
JWT_SECRET=<랜덤 문자열 32자 이상>

# SMTP (이메일 발송)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=<Gmail 주소>
SMTP_PASS=<Gmail 앱 비밀번호>
SMTP_FROM=<발신자 이름 <email>>

# 소셜 로그인 (선택)
NAVER_CLIENT_ID=<네이버 로그인 API>
NAVER_CLIENT_SECRET=
KAKAO_CLIENT_ID=<카카오 로그인 API>
KAKAO_CLIENT_SECRET=

# 토스페이먼츠 (결제, 선택)
TOSS_CLIENT_KEY=<테스트키>
TOSS_SECRET_KEY=<테스트 시크릿키>

# 환경
NODE_ENV=development
```

### 1.3 서비스 시작

```bash
# 전체 서비스 빌드 및 시작 (최초 실행 시 5~10분 소요)
docker compose up -d --build

# 정상 기동 확인
docker compose ps

# 브라우저에서 접속
# http://localhost:4001
```

### 1.4 서비스 중지 / 재시작

```bash
# 전체 중지
docker compose down

# 전체 재시작 (코드 변경 후)
docker compose up -d --build

# 특정 서비스만 재시작
docker compose restart node-server
docker compose restart client
```

## 4. DB 스키마 (전체 24개 테이블)

#### apartments — 아파트 기본정보
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PK | |
| name | VARCHAR(200) | 아파트명 |
| address | VARCHAR(500) | 지번 주소 |
| road_address | VARCHAR(500) | 도로명 주소 |
| lat / lng | NUMERIC(10,7) | 위도 / 경도 |
| build_year | INTEGER | 건축년도 |
| total_units | INTEGER | 총 세대수 |
| dong_count | INTEGER | 총 동수 |
| kapt_code | VARCHAR(20) | 공동주택 코드 |
| created_at | TIMESTAMPTZ | |

#### trade_history — 실거래 이력
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PK | |
| apartment_id | INTEGER FK | apartments.id 참조 |
| trade_date | DATE | 거래일 |
| price | BIGINT | 거래가 (만원) |
| floor | INTEGER | 층수 |
| area | NUMERIC(10,2) | 전용면적 (m²) |
| trade_type | VARCHAR(10) | sale/jeonse/monthly |
| dong | VARCHAR(50) | 동 이름 |
| created_at | TIMESTAMPTZ | |

> 유니크 제약: (apartment_id, trade_date, price, floor, area, trade_type)

#### commercial_properties — 상가/오피스텔 기본정보

|------|------|------|

| name | VARCHAR(200) | 건물명 |

```

### 9.2 Nginx 라우팅 규칙

```
/          → client:3000    (React SPA)

/analysis  → python-server:8000 (분석 API)

```

# 전체 재빌드

# 로그 실시간 확인
docker compose logs -f node-server

# DB 접속 (psql)
docker compose exec postgres psql -U postgres -d real_estate

# 프론트 패키지 추가 후

```

## 11. 개발 컨벤션

### 11.1 코드 스타일

**Backend (Node.js)**
- API 라우트 패턴: `router.get('/path', auth, checkPlan('basic'), async (req, res) => { ... })`
- DB 쿼리: SQL alias로 camelCase 변환 → 프론트에서 그대로 사용
- 에러 응답: `{ error: '한국어 메시지' }`

**Frontend (React)**
- 페이지: `pages/*.jsx`
- 상태: Zustand (전역), TanStack Query (서버 상태)
- API: `api/*.js` 에서 axios 호출 분리
- 스타일: TailwindCSS 유틸리티 클래스

**Backoffice (Python)**
- 모델: DB 컬럼명 그대로 (snake_case)
- 라우터: FastAPI 데코레이터

### 11.2 커밋 메시지

```
<type>: <한국어 설명>

타입: feat / fix / refactor / docs / chore
예시: feat: 경매 필터 추가
```

### 11.3 금지사항

- `.env` 파일 커밋 금지
- `console.log` 디버깅 코드 남기지 않기
- DB 스키마 변경 시 반드시 기존 데이터 영향 검토
- `postgres/init.sql`이 DB source of truth — 여기서 테이블 구조 확인
